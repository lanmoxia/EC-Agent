# 活状态锚点协议（最高优先级）

你**每一次最终回复**（交还控制权给用户的那条消息）的**第一行**，必须且只能是这样一行活状态锚点，格式固定、字段顺序不变：

```
ANCHOR: lanmoxia | current_task=<本轮在做的具体任务> | changed=<本轮真实改动> | next=<下一步具体动作>
```

四个字段的硬要求：

- `lanmoxia`：固定锚点词，原样输出，不可省。
- `current_task`：本轮正在做的**具体**任务。禁止空话（如 `优化`/`处理`/`当前任务`/`继续`）。
- `changed`：本轮**真实发生**的改动，必须对应实际操作——动了哪个文件、跑了什么命令、定了什么决策、做了什么验证。
  - 若本轮确实没改文件（纯讨论/检索），写清楚做了什么，例：`未改文件，检索官方文档核实X`。
  - 禁止只写 `无`/`已完成`/`按要求处理`/`优化完成` 这类没有具体内容的词。
- `next`：下一步**具体**动作。禁止 `继续优化`/`后续处理`/`等待指示` 这类空话。

这一行之后，正常输出回复正文。

> 这条锚点由 `.claude/hooks/anchor-check.js`（Stop 钩子）自动校验：缺锚点、含空话、changed 对不上本轮真实操作、或与用户本轮请求完全不相关，都会被打回重答，并记入 `.claude/anchor-violations.log`。它取代了旧的"每句输出 lanmoxia"规则。
