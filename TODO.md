## 下一大方向：视频分析→生成完整工程（优先级最高）

这是一个系统性工程，待专项讨论后拆分实施。核心目标是把"分析→提示词→生视频→验收→下一镜"跑成一条完整闭环。

### 三个子方向

#### 1. 视频分析报告质量提升
- 报告对"复刻"的实际帮助有多大？哪些段落有用/没用？
- 如何让9段报告对生视频提示词更直接可用
- 角色外观档案精度（服装逐件/体型/体态细节）
- 第一帧构图描述的准确率（当前 L3 已验证左右常颠倒）

#### 2. 豆包提示词更精准
- 当前3版策略实测效果如何？用户偏好哪版？
- 提示词哪些维度影响最大（构图/台词/服装/镜头运动）
- 规则提炼：从 prompt-experiments.md 积累的实验记录里提炼新规则写入 CLAUDE.md

#### 3. LibTV 完整生产工作流（最重）
- **原视频抽帧**：提取场景帧/角色帧/道具帧 → 作为参考图上传 LibTV
- **角色主体上传**：libtv upload + 节点绑定
- **音色创建**：TTS 配音工作流（Minimax Speech 等）
- **合理分镜**：根据分析报告§7（镜头时序）+ L1 detectScenes 切点 → 决定每镜时长
- **逐镜生成**：每镜 prompt + 参考图 → libtv node --run
- **验收逻辑**：生视频质量怎么评估？能否接受？对比分析（现有 compare 端点）
- **帧接续**：下一镜以上一镜最后帧为起始（libtv download → 抽最后帧 → 上传到下一镜节点）
- **整体拼接**：所有镜拼成完整视频
